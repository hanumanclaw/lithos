## Codebase Patterns
- KnowledgeMetadata fields: add to dataclass, `_KNOWN_METADATA_KEYS`, `to_dict()`, and `from_dict()` — all four must stay in sync
- Conditional serialization: use `if self.field` for list fields (omit empty), `if self.field is not None` for optional str fields
- Follow `source_url` as the pattern for adding new metadata fields (conditional serialization, from_dict default, round-trip tests)
- Tests use `knowledge_manager` fixture from conftest.py; `test_config` gives access to temp storage paths
- `WriteResult` is the standard return type for create/update — always check `result.status` and `result.document`
- `validate_derived_from_ids()` uses `uuid.UUID()` for validation/normalization — returns sorted, deduplicated, lowercased list
- Ruff B904: always use `raise ... from err` in except clauses
- Provenance indexes: `_doc_to_sources` (forward), `_source_to_derived` (reverse), `_unresolved_provenance` (missing refs), `_id_to_title` (title cache)
- Two-pass scan: pass 1 collects all doc IDs + provenance pairs into `deferred_provenance`, pass 2 classifies resolved/unresolved — avoids scan-order artifacts
- For tests creating raw docs on disk, use `frontmatter.Post()` + `fm.dumps()` to write valid markdown files with frontmatter
- `create()` accepts `derived_from_ids: list[str] | None = None` — validates before any side effects, updates all provenance indexes, auto-resolves previously-unresolved refs, returns warnings for missing sources
- `update()` derived_from_ids semantics: `_UNSET`=preserve, `None`/`[]`=clear, non-empty list=replace — uses `_remove_provenance_entries()` helper to clean up old reverse indexes before applying new ones
- `_remove_provenance_entries(doc_id)` is a helper that removes a doc from `_source_to_derived` and `_unresolved_provenance` based on its `_doc_to_sources` — reusable for update and delete paths
- MCP boundary mapping for `derived_from_ids` in `lithos_write`: `None`=preserve (→`_UNSET`), `[]`=clear, non-empty=replace — follows same pattern as `source_url`
- `to_dict()` omits empty `derived_from_ids` — integration tests checking cleared provenance should use `.get("derived_from_ids", [])` not direct key access
- `sync_from_disk(path)` is the manager-owned API for external file changes — handles new + modified uniformly, updates all indexes including provenance, auto-resolves unresolved refs
- `handle_file_change` in server.py: delete → `manager.delete()`, non-delete → `manager.sync_from_disk()` then pass doc to search/graph
- `_rebuild_indices()` must call `self.knowledge._scan_existing()` to rebuild provenance indexes alongside search/graph
- `lithos_provenance` BFS: "sources" walks `_doc_to_sources`, "derived" walks `_source_to_derived`; `_bfs_provenance()` helper on `LithosServer`
- Ruff SIM114: combine `if`/`elif` branches that do the same action using `or`
